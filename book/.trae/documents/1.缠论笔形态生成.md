# 1. 缠论笔形态生成

## 一、模块定位

缠论模块位于 **C. 股价骨架计算层**，属于项目核心业务逻辑，放在 `modules/chanlun/` 下。负责从原始K线数据中提取股价的结构化骨架（分型、笔），为上层蔡森形态识别提供输入。同时支持多级别分析，结果通过 `lib/db/` 缓存到 MongoDB。

**依赖关系**：
- 数据输入：来自 `lib/data_source/` 的 K线 DataFrame
- 缓存读写：通过 `lib/db/cache_manager.py` 操作 MongoDB
- 不依赖其他 modules 子模块

## 二、数据流

```
lib/data_source/data_fetcher.py
    ↓ DataFrame [trade_date, open, high, low, close, volume, ...]
modules/chanlun/include_processor.py
    ↓ DataFrame (经过包含关系处理的K线)
modules/chanlun/fractal_detector.py
    ↓ List[Fractal] (顶分型/底分型序列)
modules/chanlun/bi_generator.py
    ↓ List[Bi] (笔序列)
modules/chanlun/multi_level_analyzer.py
    ↓ Dict[period, List[Bi]] (多级别笔序列，通过 lib/db/ 缓存)
```

## 三、K线包含关系处理

### 3.1 文件：`modules/chanlun/include_processor.py`

### 3.2 包含关系定义
一根K线的高低点全在另一K线的范围内：
- 包含：`k1.high >= k2.high and k1.low <= k2.low` 或 `k2.high >= k1.high and k2.low <= k1.low`

### 3.3 合并方向判断
- **向上合并**：当当前趋势方向向上时（当前K线收盘价 > 前一根非包含K线收盘价）
  - 新K线：`high = max(k1.high, k2.high)`, `low = max(k1.low, k2.low)`
- **向下合并**：当当前趋势方向向下时（当前K线收盘价 < 前一根非包含K线收盘价）
  - 新K线：`high = min(k1.high, k2.high)`, `low = min(k1.low, k2.low)`

### 3.4 处理流程
```
输入: 原始K线 DataFrame (N行)
输出: 处理后K线 DataFrame (M行, M <= N)

算法:
1. 初始化 direction = None, result = []
2. 遍历每根K线:
   a. 如果 result 为空，直接加入
   b. 判断当前K线与 result[-1] 是否存在包含关系
   c. 若存在包含关系:
      - 若 direction 为 None: 根据前一根K线的涨跌确定 direction
      - 按 direction 进行合并，替换 result[-1]
   d. 若不存在包含关系:
      - 根据 close 与 result[-1].close 的关系更新 direction
      - 加入 result
3. 返回 result DataFrame
```

### 3.5 数据结构
```python
@dataclass
class ProcessedKLine:
    index: int          # 原始K线索引
    trade_date: str     # 交易日期
    open: float
    high: float
    low: float
    close: float
    volume: float       # 合并时取两根K线成交量之和
    original_indices: List[int]  # 被合并的原始K线索引列表
```

### 3.6 缓存策略
- 包含处理结果是中间数据，**不单独持久化**到 MongoDB
- 每次从原始K线数据实时计算（O(n)复杂度，性能足够）
- 若后续发现性能瓶颈，可考虑缓存

## 四、分型识别

### 4.1 文件：`modules/chanlun/fractal_detector.py`

### 4.2 分型定义
基于包含关系处理后的K线序列，取连续三根K线：
- **顶分型**：中间K线高点为三者最高，低点也为三者最高
  - `k[i-1].high < k[i].high > k[i+1].high` 且 `k[i-1].low < k[i].low > k[i+1].low`
- **底分型**：中间K线低点为三者最低，高点也为三者最低
  - `k[i-1].low > k[i].low < k[i+1].low` 且 `k[i-1].high > k[i].high < k[i+1].high`

### 4.3 算法实现
```python
@dataclass
class Fractal:
    fractal_type: str       # "top" / "bottom"
    trade_date: str         # 中间K线日期
    high: float             # 分型最高价
    low: float              # 分型最低价
    kline_index: int        # 处理后K线序列中的索引
    original_indices: List[int]  # 对应的原始K线索引

def detect(processed_klines: List[ProcessedKLine]) -> List[Fractal]:
    """
    滑动窗口遍历，时间复杂度 O(n)
    """
    fractals = []
    for i in range(1, len(processed_klines) - 1):
        prev, curr, next_k = processed_klines[i-1], processed_klines[i], processed_klines[i+1]
        if (curr.high > prev.high and curr.high > next_k.high and
            curr.low > prev.low and curr.low > next_k.low):
            fractals.append(Fractal("top", curr.trade_date, curr.high, curr.low, i, curr.original_indices))
        elif (curr.low < prev.low and curr.low < next_k.low and
              curr.high < prev.high and curr.high < next_k.high):
            fractals.append(Fractal("bottom", curr.trade_date, curr.high, curr.low, i, curr.original_indices))
    return fractals
```

### 4.4 缓存策略
- 分型数据持久化到 MongoDB `chanlun_fractions` collection
- 缓存键：`{ts_code, period, trade_date}`
- 通过 `lib/db/cache_manager.py` 进行缓存读写
- 增量更新：删除最新缓存日期之后的所有分型，重新计算

## 五、笔生成算法

### 5.1 文件：`modules/chanlun/bi_generator.py`

### 5.2 笔的定义
笔是连接两个相邻顶底分型的价格运动：
- 向上笔：从底分型到顶分型
- 向下笔：从顶分型到底分型

### 5.3 笔的生成条件
1. 必须是相邻的顶分型和底分型（交替出现）
2. 顶底分型之间至少有 1 根独立K线（非构成分型的K线）
3. 考虑包含关系处理后，顶底之间至少有 3 根K线
4. 新笔定义：从最低到最高之间至少有 4 根K线（包含处理后）

### 5.4 算法实现
```python
@dataclass
class Bi:
    bi_id: int                 # 笔序号
    direction: str             # "up" / "down"
    start_date: str            # 起始日期
    end_date: str              # 结束日期
    start_price: float         # 起始价
    end_price: float           # 结束价
    start_fractal: Fractal     # 起始分型
    end_fractal: Fractal       # 结束分型
    kline_count: int           # 包含的原始K线数量
    price_change: float        # 价格变动
    price_change_pct: float    # 价格变动百分比

def generate(fractals: List[Fractal], processed_klines: List[ProcessedKLine]) -> List[Bi]:
    """
    遍历分型序列，识别有效笔
    """
    bis = []
    if len(fractals) < 2:
        return bis

    filtered = _filter_consecutive_fractals(fractals)

    i = 0
    while i < len(filtered) - 1:
        f1, f2 = filtered[i], filtered[i + 1]

        if f1.fractal_type == f2.fractal_type:
            i += 1
            continue

        gap = abs(f2.kline_index - f1.kline_index) - 1
        if gap < 1:
            i += 1
            continue

        if f1.fractal_type == "bottom" and f2.fractal_type == "top":
            direction = "up"
        elif f1.fractal_type == "top" and f2.fractal_type == "bottom":
            direction = "down"
        else:
            i += 1
            continue

        kline_count = _count_original_klines(f1, f2, processed_klines)
        bi = _build_bi(bis, direction, f1, f2, kline_count)
        bis.append(bi)
        i += 1

    return bis
```

### 5.5 相邻同类型分型处理
当出现连续两个顶分型（或底分型）时，保留更极端的那个：
- 连续顶分型：保留 high 更高的
- 连续底分型：保留 low 更低的

### 5.6 笔的确认与更新
- **已确认笔**：起始和结束分型都已被后续分型"离开"（中间至少隔了一个反向分型）
- **未确认笔**：最后一个分型可能随新K线变化
- 缓存时只持久化**已确认笔**，最后一笔标记为 `confirmed: false`

### 5.7 缓存策略
- 笔数据持久化到 MongoDB `chanlun_bis` collection
- 缓存键：`{ts_code, period, bi_id}`
- 通过 `lib/db/cache_manager.py` 进行缓存读写
- 增量更新逻辑：
  1. 查询 MongoDB 中该股票+级别的最新已确认笔
  2. 从最新已确认笔的结束分型开始，向后重新计算
  3. 删除该日期之后的未确认笔，写入新的计算结果

## 六、多级别分析

### 6.1 文件：`modules/chanlun/multi_level_analyzer.py`

### 6.2 支持的级别
| 级别 | period 参数 | 数据来源 |
|------|-------------|----------|
| 5分钟 | `5min` | BaoStock 分钟数据 |
| 30分钟 | `30min` | BaoStock 分钟数据 |
| 日线 | `daily` | Tushare/BaoStock |
| 周线 | `weekly` | Tushare/BaoStock |
| 月线 | `monthly` | Tushare/BaoStock |

### 6.3 级别递归规则
- 三个5分钟走势重叠 → 构成一个30分钟中枢
- 三个30分钟走势重叠 → 构成一个日线中枢
- **初版实现**：各级别独立并行处理，不做递归构建（递归构建复杂度高，留作后续优化）

### 6.4 多级别并行分析
```python
from lib.data_source.data_fetcher import DataFetcher
from lib.db.cache_manager import CacheManager

class MultiLevelAnalyzer:
    def __init__(self, cache_manager: CacheManager, config: dict):
        self.cache = cache_manager
        self.config = config
        self.periods = config.get("chanlun", {}).get("periods", ["daily", "weekly", "monthly"])

    def analyze(self, symbol: str, periods: List[str] = None) -> Dict[str, List[Bi]]:
        """
        并行分析多个级别的笔序列
        """
        periods = periods or self.periods
        results = {}

        with ThreadPoolExecutor(max_workers=min(len(periods), 4)) as executor:
            futures = {
                executor.submit(self._analyze_single_level, symbol, period): period
                for period in periods
            }
            for future in as_completed(futures):
                period = futures[future]
                results[period] = future.result()

        return results

    def _analyze_single_level(self, symbol: str, period: str) -> List[Bi]:
        """
        单级别分析：包含处理 → 分型识别 → 笔生成
        结果通过 lib/db/cache_manager.py 缓存到 MongoDB
        """
        cached_bis = self._load_bis_from_cache(symbol, period)
        if cached_bis is not None:
            return cached_bis

        with DataFetcher() as fetcher:
            df = fetcher.get_stock_data(symbol, data_type=period)

        if df.empty:
            return []

        processed = IncludeProcessor.process(df)
        fractals = FractalDetector.detect(processed)
        self._save_fractals_to_cache(symbol, period, fractals)
        bis = BiGenerator.generate(fractals, processed)
        self._save_bis_to_cache(symbol, period, bis)

        return bis
```

## 七、增量更新机制

### 7.1 触发条件
- 新交易日到来，获取到新K线数据后
- 手动触发重新分析

### 7.2 更新流程
```python
def incremental_update(self, symbol: str, period: str):
    """
    增量更新指定股票+级别的缠论分析
    """
    latest_bi = self.cache.find_analysis_cache(
        "chanlun_bis",
        {"ts_code": format_code(symbol), "period": period},
        sort_key="bi_id"
    )

    if latest_bi.empty:
        return self._analyze_single_level(symbol, period)

    latest_end_date = latest_bi.iloc[-1]["end_date"]

    with DataFetcher() as fetcher:
        df = fetcher.get_stock_data(
            symbol, data_type=period,
            start_date=latest_end_date
        )

    if df.empty:
        return self._load_bis_from_cache(symbol, period)

    self.cache.invalidate_cache_after("chanlun_bis", format_code(symbol), period, latest_end_date)

    confirmed_end = self._get_last_confirmed_date(symbol, period)
    full_df = fetcher.get_stock_data(symbol, data_type=period, start_date=confirmed_end)

    processed = IncludeProcessor.process(full_df)
    fractals = FractalDetector.detect(processed)
    bis = BiGenerator.generate(fractals, processed)
    self._save_fractals_to_cache(symbol, period, fractals, replace_after=confirmed_end)
    self._save_bis_to_cache(symbol, period, bis, replace_after=confirmed_end)
```
